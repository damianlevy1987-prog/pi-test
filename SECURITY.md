# Security Policy

## Supported Versions

Security fixes are currently provided for the latest `master` branch and latest release tag.

## Reporting a Vulnerability

If you discover a vulnerability:

1. **Do not open a public issue with exploit details.**
2. Email the maintainer with:
   - affected component/path
   - impact assessment
   - reproduction steps
   - suggested remediation (if known)
3. You will receive acknowledgement within **72 hours**.
4. A remediation plan/ETA will be shared after triage.

## Disclosure

- Coordinated disclosure is preferred.
- Public disclosure should wait until a fix or mitigation is available.

## Scope

This policy covers API auth, token lifecycle, RBAC, request validation, rate-limiting, and data persistence components in this repository.
